INNER JOIN returns rows with matching values in both tables.

LEFT JOIN returns all rows from the left table and matching rows from the right table.

RIGHT JOIN returns all rows from the right table and matching rows from the left table.

FULL OUTER JOIN returns all matching and non-matching rows from both tables.

GROUP BY aggregates rows sharing common column values.

HAVING filters aggregated results after GROUP BY.

WHERE filters rows before aggregation.

Window functions perform calculations across related rows without collapsing them.

ROW_NUMBER assigns unique sequential numbers to rows.

RANK assigns ranks while allowing gaps for ties.

DENSE_RANK assigns ranks without gaps.

Common Table Expressions improve query readability and modularity.

Indexes improve query performance by speeding up lookups but may slow inserts and updates.

Normalization reduces redundancy and improves data consistency.

Denormalization improves read performance by intentionally duplicating data.

Primary keys uniquely identify rows in a table.

Foreign keys maintain referential integrity between tables.

Partitioning splits large tables into smaller logical pieces to improve performance.

Execution plans help analyze SQL query performance and optimization opportunities.
