An ML pipeline typically includes data ingestion, preprocessing, feature engineering, training, evaluation, deployment, monitoring, and retraining.

Batch inference processes data at scheduled intervals, while online inference serves predictions in real time with low latency.

Feature stores centralize feature computation and storage to ensure consistency between training and inference environments.

Model drift occurs when data distributions change over time, reducing model performance. Drift monitoring is essential in production systems.

Concept drift refers to changes in relationships between features and labels. Data drift refers to changes in input data distribution.

A training-serving skew occurs when training data differs from production inference data.

Latency is a key metric in online ML systems. Low-latency systems often require optimized models, caching, and scalable infrastructure.

Scalability in ML systems involves handling increasing workloads efficiently. Common approaches include horizontal scaling, distributed systems, and asynchronous processing.

Model versioning allows teams to track, deploy, and roll back different model versions safely.

Canary deployments gradually release models to small user groups before full rollout.

A/B testing compares multiple models or product variants using controlled experiments.

Monitoring metrics in production may include latency, throughput, error rate, prediction distribution, and business KPIs.

Retrieval-Augmented Generation combines information retrieval with language models. Relevant documents are retrieved first and passed into an LLM as context.

Vector databases store embeddings and enable semantic similarity search. Popular vector databases include Pinecone, Weaviate, Milvus, and FAISS.

Chunking is the process of splitting large documents into smaller semantic sections for embeddings and retrieval.

Metadata improves retrieval quality by attaching additional information such as source, author, or document type.

Reranking improves retrieval precision by rescoring retrieved documents using a secondary model.

Hallucination occurs when an LLM generates inaccurate or unsupported information. Grounding models using retrieval helps reduce hallucinations.

Caching can significantly reduce inference costs and latency in LLM applications.

Prompt engineering involves designing prompts that guide LLM behavior effectively.

Fine-tuning modifies model weights using domain-specific datasets, while prompting relies on pretrained knowledge without weight updates.
